Custom Packages

Recap: ¿Qué es un Módulo?

Un simple archivo de Python (.py) que contiene código reutilizable (funciones, clases, etc.).

2 / 17

¿Qué pasa si mi proyecto crece?

Tener todo en un solo archivo o en muchos archivos sueltos se vuelve caótico. Necesitamos una forma de agrupar módulos relacionados.

3 / 17

¿Qué es un Paquete (Package)?

En Python, un paquete es simplemente una carpeta que contiene módulos y un archivo especial llamado __init__.py.

4 / 17

El Archivo __init__.py

Este archivo le dice a Python: "Esta carpeta no es una carpeta cualquiera, trátala como un paquete". Puede estar vacío, y a menudo lo está.

5 / 17

Ejemplo: Estructura de un Paquete

Imaginemos un paquete para utilidades.

mi_proyecto/
|-- main.py
|-- mi_paquete/
    |-- __init__.py      # Le dice a Python que 'mi_paquete' es un paquete
    |-- matematicas.py
    |-- saludos.py
6 / 17

Contenido de matematicas.py

# mi_paquete/matematicas.py
def sumar(a, b):
    return a + b

def restar(a, b):
    return a - b
7 / 17

Contenido de saludos.py

# mi_paquete/saludos.py
def hola(nombre):
    return f"Hola, {nombre}"

def adios(nombre):
    return f"Adiós, {nombre}"
8 / 17

Usar el Paquete: Importación Absoluta

Desde `main.py`, usamos la notación de punto para acceder a los módulos dentro del paquete.

# main.py
from mi_paquete import matematicas
from mi_paquete.saludos import hola

print(matematicas.sumar(5, 3)) # 8
print(hola("Mundo"))           # Hola, Mundo
9 / 17

Subpaquetes

Un paquete puede contener otros paquetes (subcarpetas con su propio `__init__.py`). Esto permite organizar aún más el código.

10 / 17

Estructura con Subpaquetes

mi_proyecto/
|-- main.py
|-- mi_paquete/
    |-- __init__.py
    |-- matematicas.py
    |-- utilidades/
        |-- __init__.py # Convierte a 'utilidades' en un subpaquete
        |-- strings.py
11 / 17

Importar desde un Subpaquete

La notación de punto se extiende para navegar por la jerarquía.

# main.py
from mi_paquete.utilidades.strings import a_mayusculas

print(a_mayusculas("hola")) # HOLA
12 / 17

Comunicación Interna: Importaciones Relativas

¿Qué pasa si un módulo dentro del paquete necesita usar otro módulo del mismo paquete?

13 / 17

Ejemplo de Importación Relativa

Imaginemos que `matematicas.py` necesita una función de `saludos.py`. Se usa un punto (`.`) para indicar "desde el mismo paquete".

# En mi_paquete/matematicas.py

from .saludos import hola # '.' se refiere a 'mi_paquete'

def sumar_y_saludar(a, b, nombre):
    resultado = a + b
    saludo = hola(nombre)
    return f"{saludo}. El resultado es {resultado}."
14 / 17

Uso Avanzado de `__init__.py`

Puedes usar `__init__.py` para "promover" funciones y hacerlas más accesibles para el usuario de tu paquete.

15 / 17

Simplificando Importaciones

Al añadir código a `__init__.py`, puedes acortar la ruta de importación.

# En mi_paquete/__init__.py
from .matematicas import sumar
from .saludos import hola

# Ahora, en main.py, la importación es más corta:
from mi_paquete import sumar, hola

print(sumar(1, 2))
print(hola("Ana"))
16 / 17

Conclusión

Los paquetes son la forma estándar de organizar cualquier aplicación de Python que crezca más allá de unos pocos archivos. Permiten crear librerías reutilizables, lógicas y fáciles de mantener.
